# Exercises – Day 2

## Part 1 – AI-powered edge cases

1. **Additional edge cases**  
   Review `tests/examples/day2/login-edge-cases.spec.ts` and identify additional edge cases that are not yet covered. Use AI to generate tests for at least 2-3 additional edge cases (e.g., empty username/password, special characters, very long inputs). Use the existing Page Object Model classes from `tests/pages/`. Save the generated tests as `tests/exercises/day2/additional-edge-cases.spec.ts`.  
   **Reference:** See `tests/exercises/day2/additional-edge-cases.spec.ts` for example implementation.

## Part 2 – Playwright agents (chat)

1. **Playwright agents on a different application**  
   Choose a different web application (e.g., [The Internet](https://the-internet.herokuapp.com/), [DemoQA](https://demoqa.com/), or any other public testing site). Use Playwright agents (Planner → Generator → Healer) in VS Code Insiders to generate test cases for that application. Document:
   - Which application you chose and why
   - The test plan generated by the Planner agent
   - The test code generated by the Generator agent
   - **Healer experiment**: Intentionally break one of the generated tests (e.g., change a selector, modify an assertion, or introduce a timing issue). Use the Healer agent to suggest a fix. Document what you broke, the fix suggested by the Healer agent, and a summary of the diff/changes.

   Save your documentation and generated tests in `tests/exercises/day2/agents-<your-name>/` directory.
